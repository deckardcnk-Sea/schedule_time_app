## 一、实现架构

| 能力 | 实现方式 | 代码位置 |
|---|---|---|
| 后台持续计时 | `flutter_background_service` 前台服务，App 退后台/锁屏仍用真实墙钟时间算 elapsed | `lib/services/background_timer_service.dart` |
| 系统悬浮窗 | `system_alert_window` 全局悬浮计时条（活动名 + 实时计时） | `lib/services/floating_timer_window.dart` |
| UI 桥接 | 开始/暂停/继续/停止时统一驱动服务+悬浮窗；从后台回前台重新同步 | `lib/pages/quick_record_page.dart`（`_syncBackground` / `didChangeAppLifecycleState`） |
| 初始化 | `main()` 调用 `BackgroundTimerService.initialize()` | `lib/main.dart` |

## 二、真机试验步骤

1. 编译 APK：`flutter build apk --debug`
2. 安装到 Android 真机（USB 调试或 `flutter install`）。
3. 打开 App → 底部「快速记录」页 → 点任一活动开始计时。
4. **授权悬浮窗**：首次开始计时会弹出系统「显示在其他应用上」授权；允许后顶部出现悬浮计时条。
   - 若拒绝：会弹提示框引导去系统设置开启，计时仍在通知栏可见。
5. **验证后台计时**：按 Home 键退到后台 / 锁屏 → 下拉状态栏可见「时间记录」通知持续走表；若已授权，悬浮条也持续显示。
6. **验证恢复**：回到 App，计时 UI 与服务状态一致（生命周期监听重新同步）。
7. 点「停止并保存」→ 二次确认 → 写入日历；通知 + 悬浮窗自动消失。

## 三、关键权限（已在 AndroidManifest 声明）

- `FOREGROUND_SERVICE` / `FOREGROUND_SERVICE_DATA_SYNC`：前台服务保活
- `SYSTEM_ALERT_WINDOW`：系统悬浮窗（需用户手动授权）
- `POST_NOTIFICATIONS` / `WAKE_LOCK`：通知与唤醒锁

## 四、已知限制（重要）

1. **悬浮窗样式受限**：`system_alert_window` 2.x 仅接受 `notificationTitle` + `notificationBody` 文本字段，
   渲染为系统样式悬浮条（非旧版自定义彩色圆角窗）。Android 11+ 默认走 Bubble 气泡模式，
   需用户在「设置 → 应用 → 通知 → 气泡」开启「所有对话可气泡」。
2. **杀进程即停**：需求4 的"后台"指 App 退到后台/锁屏；若用户从最近任务**强杀 App 进程**，
   前台服务与悬浮窗会随进程终止（Android 不允许第三方 App 在进程被杀后继续运行，除非用系统闹钟/WorkManager 周期性校正——本次未做）。
3. **iOS / Web 不支持**：iOS 无等价全局悬浮窗 API；Web 无原生后台/悬浮窗，需求4 仅 Android 真机有效。
4. **前台服务通知不可隐藏**：Android 要求前台服务必须有常驻通知，这是"后台计时"的可见载体之一（与悬浮窗互补）。

## 五、如需进一步增强（本次未做）

- 强杀后继续计时：改用 Android `AlarmManager` / `WorkManager` 周期性校正（牺牲"连续计时精度"）。
- 自定义彩色悬浮窗：降级 `system_alert_window` 到 1.x，或自写原生 `WindowManager` 悬浮 View。
- 点击悬浮窗/通知直接回到对应活动页：在通知/悬浮窗加 PendingIntent 深链。
